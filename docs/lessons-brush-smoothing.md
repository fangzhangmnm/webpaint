# 经验：笔刷平滑重写（quad → EMA → SmoothDamp）

> as-of v245 / 2026-06-13。配套施工图 `brush-procreate-smoothing.md`、台架 `bench/smoothing-bench.mjs`。
> 这篇是**为什么**（耐老化）；how 看施工图、what 看代码。给下一个 AI / 未来的我。

一次把 prod 用了 90 版的「弧长二次 WLS quad frozen/tail」整条换成 Procreate 三段式
（重采样 → SmoothDamp → 弧线 tail）的过程，踩出来的几条值钱的。

## 数学层

**1. 重采样到固定弧长 = 帧率无关，白送。**
任何因果滤波器（EMA/SmoothDamp）直接跑在 raw event 上都是帧率相关的：120Hz 比 60Hz 多喂一倍点
→ 平滑强一倍。把输入先按**固定弧长 Δ 重采样**再喂滤波器，平滑强度只取决于走过的弧长，与事件
密度无关（bench 实测 1× vs 3× 密度偏差 0.0000px）。旧 quad 也帧率无关，但靠每帧重算窗口换；
重采样是结构性免费。**凡是做手写笔平滑，先重采样。**

**2. 一阶滤波器做不出弧线收笔——这是本质，不是调参问题。**
纯一阶 EMA（`pos += (target−pos)·(1−a)`）抬笔时朝**单个固定终点**指数逼近，轨迹是 `pos→终点`
那根**弦**，必然是直的（直尾巴）。想要 Procreate 那种弧线收笔，必须**二阶**系统（带速度 `vel`
状态）：抬笔瞬间落点仍有切向动量，弹簧把它拉向终点时顺动量冲出再弯回 = 自然弧。一阶没有 `vel`，
喂常量终点只会走直线。**症状是「尾巴是直的」时，别去调阻尼/系数，先问几阶。**

**3. 因果滤波器 ⇒ 锚点落定即终值、永不回改。**
EMA/SmoothDamp 的 `pos_n` 只依赖 `(pos,vel)_{n-1}` 和 `q_n` → 一旦算出就是最终值。所以**不用
每帧重算后缀**（quad 的大头开销 + 复杂度来源）。锚点只追加、frozen 段直接烤死。这就是用户说的
「Procreate 比我们想的简单」的真正出处——不是公式简单，是**因果性消掉了重算**。

**4. 临界阻尼 SmoothDamp = body 不抖 + 有动量，但弧偏浅。**
临界阻尼（GPG4 有理近似 SmoothDamp）跟随时不过冲（body 不振铃，转角不甩），又保留速度连续性
（收笔有动量出弧）。代价：弧比真实曲线**浅**（under-bow，bench sl=0.9 约 0.8px vs 真弧 ~2px）。
要更夸张就调松阻尼（增大动量）或改「flush 真实缓冲点」。**这是产品取舍不是 bug。**

**5. 滑块→强度别直取，按目标量反解。**
EMA 的滞后对系数 `a` 在 `a→1` 处极敏感，`a = streamline` 直取会让滑块「前 90% 没感觉、最后 10%
暴冲」。按**目标滞后长度** `L` 反解（EMA: `a=L/(L+Δ)`；SmoothDamp: `T=lag/Δ`）→ 滑块线性可用。
标定（`streamlineMaxLagPx`）定义「滑块 1.0 = 多少 px 滞后」，是全局一份的天花板。

## 架构层

**6. 贴笔尖 + 弧 tail = 已提交锚点串 ⊕ transient tail，finish = 把 tail 转正。**
`cx = [已提交锚点(因果终值，烤死) … transient tail(每 push 重建，末点钉光标)]`，`frozenIndex =
提交数−1`。**画途中**的 tail 就是「从带动量落点 flush 到光标的那段弧」（每 push 用 pos/vel 的
拷贝非破坏重算）；**抬笔 finish() 只是把当前这段预览弧整段转正**（`_committed = count`，不重算、
点不动）→ **预览所见即所得，结构上必然一致**。想让预览=最终，就让它俩是同一段代码。

**7. 换核不换壳：保住能用的渲染管线。**
brush.js 的 frozen/tail 双 buffer/overlay/dirty-rect **渲染**机制是好的，三次迭代（quad→EMA→
SmoothDamp）全程没动它，只换 `stroke-smoother.js` 的平滑**核** + 一个 overlay 缓存键（count→seq，
因为慢速不落新锚点时 count 不变但笔尖动了、tail 仍要刷）。深模块的接口对了，换实现是局部的。

## 过程 / 产品层

**8. 「架空旋钮」陷阱：schema 里有 ≠ 真的接线了。**
`stabilization` 在 brush settings + 默认笔里躺了 ~90 版，但主笔刷路径**只接了 streamline**，
stabilization/motionFilter 对画笔**完全没生效**。用户「以为在调的旋钮根本没连上」。教训：参数
在设置面板里出现，不代表它真的接到了引擎；**dev 面板让你把一个值 ×100 看有没有变化，就是专门
抓这种死参数的**（×100 没反应 = 没接线）。删旋钮前先确认它是不是早就架空了。

**9. 改「存储值的含义」必须迁移默认值，否则静默手感回归。**
把 `streamlineMaxLagPx` 24→48（滑块加倍潜力）= 所有现存 streamline 值的**含义**变了。出厂默认
笔必须同步**折半**（0.3→0.15、勾线 0.9→0.45）才能「滑块变强」同时「出厂手感不变」。**用户自己
存的笔无法自动迁移 → 必须明确告知**（他们的笔会变 2× 强）。删数据里早已无效的死值时也一样：
勾线的 `stabilization:0.9` 是架空期设的、从没被感受过，新模型会让它突然生效 → 归 0 防意外。

**10. 手感是真机终判，但能静态验的先验完。**
不在 device 上瞎调阻尼/系数。但**先写问题陈述**（输入是什么、输出是什么）、**bench 钉死能静态
验的不变量**：帧率无关(0.0000px)、笔尖滞后(恒 0)、直线收笔(直)、弯笔收笔(出弧)。这些过了再推
dev 让用户真机判「鼓度够不够、跟不跟手」。math/手感 bug 禁止猜测式调试（见总 CLAUDE.md）。

**11. 闭源行为的复刻：行为描述 + 开源反推配对。**
Procreate 闭源，官方 Handbook 只有「行为描述」（"squash 抖动朝中心线"）。靠它纠正手感方向，靠
Krita/MyPaint 开源反推算法（死区拉绳、finish line catch-up）。「为什么是直尾巴」的一阶 vs 二阶
诊断就来自用户喂进来的这份配对研究（`docs/brush proposal 20260613.md`）。

## 转角门控（保棱角）层

**12.（先错后对）保棱角别用「阈值 + 硬锚点」——会把紧弧剁成多边形。改连续映射。**
第一版（v247）：检测转角（input-dir 变化超阈值）→ 把顶点钉硬锚点。**用户实测在小半径弧上出多边形。**
根因不是「用速度还是曲率当信号」,而是更底层的**阈值化本身**：`if cosA<阈值 then 当成角` 把连续的曲率
硬切成二值,紧弧上每点的 cosA 都在阈值附近 → 逐个判成角、各自插锚 → 紧弧被剁成多边形。**只要有阈值+
锚点吸附,这问题就消不掉。**
**对的做法（v248）：全程去阈值、去锚点,改连续映射。** 把「转角」重新定义成**系统能平滑表示的最小弧
半径 R_min**——角就是 R_min 的弧,永远不是几何顶点。读三点 Menger 曲率 → `smoothstep(R_min,R_smooth,R)`
连续插值平滑强度（紧弧跟手、直线重平滑）。无锚点、无 vel 复位 → pos 连续流动 → 紧弧是平滑弧不是多边形
（bench:R=8 紧弧逐步转角 max≈avg）。**通用教训:要保留的「特征」是连续谱的一端(角=R→R_min),别用阈值
把它二值化再吸附;raw 曲率噪 → 先低通 κ 标量再映射。** 速度信号留给「顿感→线宽」,别拿来当几何门控
（直线段速度有歧义:慢直线会被误判成精细操作而少平滑 → 抖）。

**12b. 旧第一版的两条子结论仍对,但已被 12 取代:** ① 检测信号要选「与你想忽略的量(lag)无关」的——
input-dir 变化对、vel-vs-(q−pos) 在大 lag 下瞎(pos 滞后时 q−pos 仍指旧腿)。② 在一个空间尺度(span/κ低通)
上测以滤掉更细的噪声。连续映射版里这两条体现为:用 input 三点曲率(lag 无关)+ κ 低通(空间尺度去噪)。

**13. 我们没有渲染样条 = 省一半事。**
参考特别警告「就算滤波器透传了转角，渲染样条（Catmull-Rom/B 样条 C1 连续）还会把它圆回去」。我们的
引擎是**沿折线等距 stamp**，折线就是路径，没有样条再插值 → 这一处圆角对我们根本不存在。**抄实现前
先确认哪些步骤你的架构里压根没有。**
