# 0001 — 时间门控的弧长窗口平滑（time-gated arc-length stroke smoothing）

Status: Accepted · 2026-06-02

## Context

笔触位置 [[Stroke smoother]] 的演进：EMA(v40) → 四件套+速度自适应(v124) → 弧长窗口二次回归(v148, "quad")。
当前 quad 按**弧长** s 取窗 + 三角加权 + 局部二次拟合（保曲率、不内缩）。

问题：弧长参数化看不见 **[[Dwell (顿)]]**——故意停顿是高时间、近零位移，弧长几乎不累积。
实测（L 形角，进/出腿快、角处顿 D 帧）：

| 顿 D 帧 | 弧长权重磨圆(到顶点 px) | 时间权重磨圆 |
|---|---|---|
| 6 | 3.45 | 1.64 |
| 12 | 3.96 | **0.14** |
| 24 | 2.06 | **0.00** |

弧长：顿多久都照样磨圆（角在弧长里近乎不存在）；时间：顿越久角越尖。
用户证言：关 streamline 就不磨角 → 确认是平滑磨的；"必须画快才稳" = 生物力学（快=弹道开环、本就平滑；
慢=闭环纠偏+震颤、本就抖），算法治不了慢速*有意*曲线，但能治 dwell 那一块。
附带发现：`webpaint.vref`/V_REF 对主笔刷早已无效（只在 `_fourStageSmooth` 里，brush 走 quad 不碰）。

## Decision

在现有弧长窗口上加一道**硬时间门**（按真实事件时间戳，**不用帧数/fps**）：

- **取窗**：样本须同时满足 `|Δs| ≤ W`（弧长）**且** `|Δt| ≤ T`（时间）；权重仍用弧长三角权。
- **冻结**：`tip_s − s_i ≥ W` **或** `tip_t − t_i ≥ T`。
- **快速**（笔速 ≥ W/T，约 225px/s @ W=27px,T=120ms）：W 弧长内样本 `|Δt|=|Δs|/v ≤ W/v ≤ T` 全过门 →
  时间门不 bind → **与现状逐字节相同**（W 不变 → 快速手感零回归）。← 用户硬约束。
- **慢/dwell**（笔速 < W/T）：W 弧长够到的进/出腿在时间上 >T → 被门剔除 → 窗口只剩最近 T 秒(=dwell) →
  **保住尖角**；且时间到 T 自动冻结 → 慢画也按时定下来（顺带治"慢画时 tail 游动、笔尖一直抖"的 live 体验）。

估计子保持**局部二次**（保曲率/不内缩，默认）；提供"曲率保持"开关，关 → 0 阶（内缩/毛笔甩尖）。

**参数与调参**：精选 3 个（`W`=streamline / `T`=dwell, ms / `内缩开关`）expose 到 brush settings 给普通用户。
另建 **dev 面板**（藏 debug flag 后）：**能暴露的平滑魔数全部暴露**，连续参数用 **textbox（不用 slider）**、
二元参数用 **开关**。默认值 = 现有写死值。调好后按参数性质 bake 成 constant / per-brush / global。

为什么 textbox+开关 而非 slider（核心：杀"煤气灯"= 饱和假阴性）：
slider 的固定 range 可能**整段落在该参数的饱和区**，或被**其它参数的取值**推进饱和 → 怎么拖都没变化 →
误判为"死参数"。textbox 能打极端数量级值**跳出饱和区**，辨别"真死 vs 此处饱和"（如已确认死掉的 V_REF）；
二元参数用开关给确定性；**全暴露** → 可把"共参"调出饱和区以**孤立**被测参数。

## Consequences

- (+) dwell 尖角保住；快速零回归；framerate 无关（时间戳，不用 fps 做 dt）。
- (+) 调参搬到设备上拖滑块，不再为调参 commit/push；dev 面板大范围滑块让死参数自我暴露。
- (−) 多一个时间维度 + 两个参数（T、内缩）。
- (−) 慢速*连续*描线（非 dwell）平滑变轻 → 生物力学抖动更显（已知物理上限，接受）。
- (−) 每个 raw 点要带时间戳（input 已有 `ev.timeStamp`，零额外成本）。

## Alternatives

- **纯时间窗**：dwell 保得好，但快速滞后 ∝ 速度×T（橡皮筋）、快速曲线过度内缩 → 否。
- **纯弧长（现状）**：快速好，但 dwell 磨角 → 否。
- **one-euro filter**（时间域因果 EMA，业界抗抖标准）：因果=有 lag，与 lookahead/frozen-tail 冲突、
  0 阶内缩、照样磨 dwell → 否。
